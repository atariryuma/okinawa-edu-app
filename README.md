# 沖縄 教育法規ナビ — 学校・学年・学級経営の“判断の根拠”

日々の**学校・学年・学級経営**での「判断の根拠」を、すぐ引く・覚えるための、**依存ゼロの単一ページWebアプリ（PWA）**です。**管理職試験の法規対策にも**（教員採用試験の教職教養の法規・時事の補強にも）使えます。インストール不要・オフライン対応・無料。

- **🔎 横断検索** … 要点・逆引き・問題・論文をまとめて検索。問題ヒットはその場で答え＋出典を表示
- **こんな時は（逆引き）** … 「体罰」「児童虐待の通告」「感染症の出席停止」など**現場の状況→根拠条文**を即提示（実務12＋施策14件）。**📋 根拠をコピー**で起案文書に貼れる
- **要点・条文** … 要点を出典付きで。各根拠は **📖 条文/資料を読む** で e-Gov・文科省の公式ページへ直リンク
- **クイズ** … 一問一答／4択／穴埋め（赤シート）／並べ替え／対応づけ。**間隔反復(SRS)** で忘れた頃に自動再出題。**⭐付箋**で見返したい問題を集約
- **論文・面接** … 頻出テーマの「序論→本論→結論＋根拠＋自校の具体策」の型。下書き保存・自己添削・**模試（時間計測）**
- **みんなの問題** … 自作問題を作って共有。出題画面の **👍役に立った／⚠️通報** で良問が上位に・不適切は自動で非公開（→ [`community-gas/README.md`](community-gas/README.md)）
- **進捗** … 定着率・連続日数・分野別・弱点リスト・最近見た。**Googleログインで複数端末に同期**、または**ファイルでバックアップ/復元**

ログインしなくても全機能が使えます（進捗はその端末に保存）。Google連携は任意です。
関連：[使い方・About](about.html)／[利用規約](terms.html)／[プライバシー](privacy.html)

## 1. GitHub Pages で公開する

1. GitHub で新しいリポジトリを作成（Public）。
2. このフォルダ一式をアップロード（`index.html` / `sw.js` / `questions.json` / `manifest.webmanifest` / `privacy.html` / `terms.html` / `about.html` / アイコン各種）。
3. **Settings → Pages** → Source を **「Deploy from a branch」**、Branch を **`main` / `(root)`** にして **Save**。
4. 1〜2分で公開URL `https://<ユーザー名>.github.io/<リポジトリ名>/` が表示されます。
5. スマホでそのURLを開き「ホーム画面に追加」でアプリのように使えます（PWA・オフライン対応）。

> HTTPS配信のため Service Worker（オフライン）と Google ログインが動作します。

## 2. Google Drive 同期を有効にする（任意）

進捗を各自の Google ドライブの**アプリ専用フォルダ（appDataFolder：他から見えない隠しフォルダ）**に保存し、スマホ⇄PCで引き継げます。OAuth クライアントID を発行します（無料）。

### 2-1. Google Cloud でクライアントIDを作る
1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを新規作成。
2. **APIとサービス → ライブラリ** で **Google Drive API** を有効化。
3. **OAuth 同意画面**：User Type は **外部(External)**、アプリ名等を入力。スコープに `https://www.googleapis.com/auth/drive.appdata` を追加。**テストユーザー**に自分を追加（公開審査は不要）。
4. **認証情報 → OAuth クライアント ID**：種類は **ウェブアプリケーション**。**承認済みの JavaScript 生成元** に公開URLの**オリジンだけ**（`https://<ユーザー名>.github.io`、パスは含めない）を追加。表示される**クライアントID**をコピー。

### 2-2. アプリに設定する
- 既定値を埋め込むなら `index.html` の `const DEFAULT_CLIENT_ID='…apps.googleusercontent.com';` に設定（このリポジトリは設定済み）。または「進捗」タブの同期カードに貼って「IDを保存」。
- **「Googleでログイン」** で許可すると、以後は自動同期（クイズ終了時・背景同期・「今すぐ同期」）。別端末でも同じアカウントでログインすれば進捗がマージされます。

> 公開リポジトリにクライアントIDが見えても、承認済みオリジンを自分のドメインに限定していれば第三者は悪用できません。

## 3. 問題の修正・追加（コンテンツ運用）

問題の**正本は `questions.json`**。**`index.html` の編集は不要**です。

1. `questions.json` を編集（1問＝1オブジェクト）。`id` は**一意かつ不変**（変えると学習進捗が孤児化）。全問に `src`（出典）。形式別フィールドは [`CLAUDE.md`](CLAUDE.md) 参照（qa/mc/cloze/order/match）。
2. **テスト＆検証**：`node test.js`（ロジック回帰）。起動時にも `selfCheck()` が id重複・出典欠落・mc範囲等を自動点検。
3. **`sw.js` の `CACHE` 名を上げる**（例 `okinawa-edu-v46` → `v47`）。← 忘れると利用者に旧版が出続けます。
4. `git add -A && git commit && git push`。GitHub Pages に自動反映。

利用者はアプリ内「✍️ 問題の作成・共有」から**5形式すべて**を自作でき、📤エクスポート／📥インポートで受け渡し、または「みんなの問題」で共有できます。

## 4. 開発・テスト

依存ゼロ・ビルドなし。**変更したら必ずテスト**：

```bash
node test.js       # index.html のロジック（SRS / マージ / 入力浄化 / 検証 …）
node test-gas.js   # community-gas/Code.gs（投稿・通報・👍の検証・自動公開判定）
# npm test でも両方実行。push/PR で GitHub Actions が自動実行（.github/workflows/ci.yml）
```

テストは**実コードを軽量スタブ上で評価**して検証します（抽出コピーではないのでテストと実装がズレません）。バグを直したら回帰ケースを `test*.js` に1つ足す運用です。設計・地雷・作法は [`CLAUDE.md`](CLAUDE.md) に集約。

## データの保存場所
- ローカル：ブラウザの `localStorage`（端末内）。
- クラウド（任意）：Google Drive の `appDataFolder` 内 `progress.json`（アプリ専用の隠しフォルダ）。
- 同期はカード単位で「最後に学習した／進んでいる方」を採用する非破壊マージ。バックアップ復元・改竄ファイルでもクラッシュ／統計汚染しないよう浄化します。

## 注意
- 問題の多くは生成AIで作成し一次資料で検証していますが、**年度依存の重点・施策や制度改正は受験年度の最新版で必ず確認**してください。各問題の「📖 条文/資料を読む」で公式情報を参照できます。
- 論文テンプレは骨子です。最新施策と自校の実態に合わせて加筆を。
